# Security policy

## Supported versions

This repository is published under the terms in [`LICENSE`](LICENSE) (educational
use). Security fixes and dependency updates are applied on a best-effort basis
on the default branch (`master`). There are no formal long-term support (LTS)
release lines for this project.

## Reporting a vulnerability

If you believe you have found a security vulnerability in Lunaz, please report
it responsibly so we can investigate and coordinate a fix before public
disclosure.

**Preferred:** Use [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
for this repository (“Report a vulnerability”) if the feature is enabled.

**Alternative:** Open an issue only if you cannot use private reporting—avoid
including exploit details or sensitive data in public text; summarize impact
and reproduction at a high level and ask maintainers to contact you for details.

Please include, where possible:

- A short description of the issue and its impact
- Steps to reproduce or proof-of-concept (without live exploitation against
  production systems you do not own)
- Affected components (e.g. `apps/backend`, `apps/web`) and versions/commits

## What to expect

- Maintainers will acknowledge receipt when possible and work toward a
  reasonable timeline for triage and remediation.
- Please allow time for review before discussing unresolved issues in public
  forums.

## Scope

Reports should concern this codebase and its documented configuration. Third-party
services, deployment environments, and dependency chains outside this repository
may need to be reported to the respective vendors.

## Safe harbor

We appreciate good-faith security research that follows this policy and avoids
harm to users, data, or systems.
